Written by Adeline Wong and Thomas Gordon

Uh oh. It looks like this puzzle doesn't have any content in it; only a disclaimer that we should look at the Library. But a quick look at the Library reveals that the "search" button has the same emoji as the title of this puzzle. This suggests that the puzzle is probably hidden in the Library's search function somehow!

If we navigate to the third floor of the library (or, if we haven't used the search function before, if we simply open the page) and search for any string, we might notice two things: firstly, that the highlighted portion of the page which includes the text we searched for has extra information appended to it; and secondly, that the rest of these pages don't seem to be fully random. In fact, each search result in the list includes the same information each time.

The search query text can overlap with the included information, so if it's too long, it'll probably overwrite most of the data; best to keep the query short (or keep requesting search results until the whole puzzle is visible). There are seven search results that come up for each query, each of which corresponds to a minipuzzle.

The puzzle consists of a wordsearch-like grid of letters and a set of enumerations. There are a few words that can be spotted in the grid: NAVIGATOR, KNIFE, and SCIENCE, for instance. But these don't seem to fit the blanks on their own.

The trick is that each of the entries are actually split into two separate parts, with one letter repeated at the end of the first half and at the start of the second half. These two parts are then hidden in the wordsearch separately. To put it another way, one of the letters in each of the entries is a 'wormhole', which 'teleports' the rest of the entry somewhere else in the grid. Each of the entries are also associated with wormholes or space-time travel in some fashion.

Author’s Notes

Thomas: This puzzle was ideated when I remarked (well, repeatedly brought up) that we had Library-related puzzles in floors one and two, but no Library puzzles in floor three at all. This presented a pacing issue: if the last time that solvers had to interact with the Library was halfway through floor two, they would spend over half the hunt completely ignoring our interesting hunt gimmick and all of the hard work that had gone into it!

The solution, then, was to create a bunch more hard work for everyone involved. (Oops. Sorry.) Over some long-forgotten Shardhunt meeting we talked over ways that a Library puzzle could work and designed the meta; then it was off to the races.

I'd like to thank my co-author, Addie, for taking my vague hand-gesturing and doing the legwork to make it happen (including all of the site tech and five of the minipuzzles).

Adeline: When we were iterating on the early drafts of this puzzle, the concept of a puzzle page containing only the "This puzzle requires use of the Library" disclaimer and nothing else was simply too good to pass up. As an added bonus, it helped us justify the multiple results on the Library search page!

Also, a few comments from me on some of the minis:

first, name these people: Alas, BRIAN DAVID GILBERT having not two but three first names disqualified him from use in this puzzle. Some pairings I like but ended up not using included JOHN OLIVER TWIST and MONTGOMERY SCOTT PILGRIM.

weve been working on these titles: This puzzle went through a few iterations, especially while trying to pick the right balance of obvious and easy-to-clue working titles. It was with much disappointment that I had to cut THE INTIMIDATION GAME (Batman Begins) and TORCHWOOD (Doctor Who) for the unfortunate truth that there are existing movies/shows that share those exact names (in the case of Torchwood, quite intentionally). I also considered, briefly, attempting to clue TEN THINGS I HATE ABOUT CLUELESS ROAD TRIPS WHEN I HARDLY CAN'T WAIT TO BE KISSED (Not Another Teen Movie), before deciding it would probably be a little bit too hard.

search your library: When I stumbled upon the idea of the "search your library" mechanic, I made a pitch for it as the minimeta for this puzzle. It's probably better that this pitch failed, as the actual minimeta is much more integrated into the Library itself, but I'm glad I got to recycle the idea as a feeder.
